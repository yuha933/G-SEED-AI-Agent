<h1>
  🌱 G-SEED 인증 항목 컨설팅 AI Agent
</h1>

<hr />

건축 설계 문서 PDF를 분석하여 녹색건축 인증기준(G-SEED)의 항목별 충족 여부, 예상 점수, 부족 점수, 개선 방향을 제안하는 문서 기반 사전검토 AI Agent 프로젝트입니다.

본 프로젝트는 실제 인증 업무에서 여러 문서에 흩어져 있는 설계 정보를 자동으로 정리하고, G-SEED 기준 DB와 연결하여 사람이 빠르게 검토할 수 있도록 돕는 것을 목표로 합니다.

## 📌 프로젝트 배경

G-SEED 인증 검토는 건축도면, 설비도면, 에너지 관련 문서, 조경계획서, 자재 인증서, 시공계획서 등 다양한 문서를 함께 확인해야 합니다.  
각 인증 항목마다 필요한 수치와 근거 문서가 다르고, 일부 항목은 OCR 결과나 도면 해석 결과를 사람이 다시 확인해야 합니다.

이 프로젝트는 이러한 과정을 다음과 같이 지원합니다.

- 문서에서 G-SEED 검토에 필요한 설계값 후보를 추출
- G-SEED 기준을 구조화된 Knowledge Base로 관리
- 항목별 충족/미달/검토 필요/근거 부족 상태를 구분
- 사람이 직접 판정 결과를 수정할 수 있는 검토 UI 제공
- 현재 예상 점수와 목표 등급까지의 부족 점수 계산
- 부족 점수를 보완하기 위한 개선 조합 추천
- 계층 탐색 RAG와 LLM을 활용한 후속 질의응답 지원

## ✨ 주요 기능

### 🧾 1. PDF 문서 업로드 및 분석

Streamlit 화면에서 다음과 같은 문서 묶음을 업로드할 수 있습니다.

- 건축도면 통합본
- 설비도면 통합본
- 에너지절약계획서 / 설계검토서
- 조경계획표 / 생태면적률 산출서
- 마감자재목록표 및 자재 인증서
- 시공계획서 / 환경관리계획서
- 운영·유지관리 / 사용자 매뉴얼
- 성능시험성적서
- 기타 참고 문서

문서는 PDF 텍스트 추출과 OCR 보조 처리를 통해 페이지 단위로 읽고, 문서 유형별 추출 규칙을 적용합니다.

### 🗂️ 2. G-SEED 기준 Knowledge Base 구축

G-SEED 2016-8 기준 중 주거용 건축물 항목을 JSON 기반 DB로 구조화했습니다.

각 항목은 다음 정보를 포함합니다.

- 전문분야
- 인증항목명
- 평가목적
- 평가방법
- 배점
- 등급 기준
- 산출식
- 필요한 설계값
- 참고 문서 및 제출서류

### ✅ 3. 항목별 사전검토 판정

추출된 설계값과 G-SEED 기준값을 비교하여 항목별 상태를 구분합니다.

| 상태 | 의미 |
|---|---|
| `pass` | 근거가 있고 기준을 충족한 항목 |
| `fail` | 근거가 있고 기준에 미달한 항목 |
| `needs_review` | OCR 또는 도면 해석 결과라 사람 검토가 필요한 항목 |
| `insufficient_evidence` | 일부 근거는 있으나 판정에 필요한 정보가 부족한 항목 |
| `not_evaluated` | 현재 문서로 평가 대상이 아니거나 판단하지 않은 항목 |

### 🧑‍💻 4. Human-in-the-loop 검토 UI

자동 추출 결과를 그대로 확정하지 않고, 사용자가 항목별로 판정 상태와 등급을 직접 수정할 수 있도록 구성했습니다.

예를 들어 `R-7.3 단위세대 환기성능 확보` 항목을 열면 해당 기준표를 확인하고, 실제 가능한 선택지 중 하나를 선택할 수 있습니다.

- 1급
- 2급
- 3급
- 4급
- 미충족
- 판단불가

이 구조는 자동화 결과를 사람이 최종 검토하는 실제 업무 흐름을 반영하기 위한 설계입니다.

### 📊 5. 예상 점수 및 개선 조합 추천

검토 결과를 바탕으로 현재 예상 점수와 목표 등급까지 부족한 점수를 계산합니다.

결과 예시는 다음과 같은 형태를 목표로 합니다.

> 현재 예상 점수는 00점이며, 목표 등급까지 00점이 부족한 상황입니다.  
> 현재 상황에서 점수를 끌어올릴 수 있는 항목들을 기준으로 몇 가지 개선 조합을 제안드립니다.

개선안은 단순히 점수가 높은 항목을 나열하는 방식이 아니라, 보완 가능성이 높은 문서와 확인해야 할 내용을 함께 제시하는 방향으로 설계했습니다.

### 🔎 6. 계층 탐색 RAG 기반 챗봇

챗봇은 단순 키워드 검색이 아니라 G-SEED 기준의 계층 구조를 따라 관련 정보를 탐색합니다.

```text
전문분야
  └─ 인증항목
       ├─ 필요 문서
       ├─ 확인해야 할 설계값
       ├─ 등급 기준
       ├─ 현재 프로젝트 판정 상태
       └─ 현재 프로젝트 추출 근거
```

예를 들어 사용자가 “생태면적률 점수를 올리려면 뭐가 필요해?”라고 질문하면, `R-6.3 생태면적률` 항목을 중심으로 필요 문서, 확인해야 할 값, 등급 기준, 관련 항목을 함께 가져와 답변합니다.

이 방식은 LLM이 질문과 관련 없는 인증 항목을 넓게 나열하는 문제를 줄이기 위한 구조입니다.

## 🔁 전체 Workflow

```text
문서 업로드
  → PDF/OCR 추출
  → 문서 유형별 설계값 후보 추출
  → G-SEED Knowledge Base 매핑
  → 항목별 판정 및 점수 계산
  → 사용자 검토/수정
  → 목표 등급 비교 및 개선안 추천
  → 챗봇 질의응답
```

## 🛠️ 기술 스택

| 구분 | 사용 기술 |
|---|---|
| Language | Python |
| UI | Streamlit |
| API | FastAPI |

## 🚀 실행 방법

### 1. 의존성 설치

```bash
pip install -r requirements.txt
```

### 2. Streamlit 실행

```bash
streamlit run gseed_agent/app.py
```

### 3. FastAPI 실행

```bash
uvicorn gseed_agent.api:app --reload
```

### 4. CLI 실행

```bash
python -m gseed_agent.main --target-score 50
```

## 🔐 LLM 설정

LLM 챗봇 기능을 사용하려면 환경변수로 API Key를 설정합니다.

```bash
set OPENAI_API_KEY=your_api_key
set OPENAI_MODEL=gpt-5.4-mini
```

API Key가 없으면 LLM 호출 없이 계층 탐색 RAG 기반 기본 답변을 반환합니다.

## 🧩 구현하면서 중점적으로 본 부분

- G-SEED 기준을 단순 텍스트가 아닌 항목별 JSON DB로 구조화
- 문서별로 추출 가능한 설계값과 평가 항목을 분리
- 자동 판정이 어려운 항목은 사람 검토 단계로 넘기는 구조 설계
- 점수 계산과 LLM 답변을 분리하여, LLM이 임의로 점수를 만들지 않도록 설계
- 단순 RAG가 아니라 인증항목, 필요 문서, 설계값, 등급 기준을 연결하는 계층 탐색 RAG 구현
- Streamlit UI와 FastAPI API를 분리하여 데모와 백엔드 확장이 모두 가능하도록 구성

## ⚠️ 한계와 개선 방향

현재 프로젝트는 실제 인증 제출을 완전히 자동화하는 시스템이 아니라, G-SEED 사전검토를 보조하는 프로토타입입니다.

주요 한계는 다음과 같습니다.

- PDF 품질과 OCR 결과에 따라 추출 정확도가 달라질 수 있음
- 도면의 공간 객체나 동선 정보는 이미지 해석만으로 완전한 구조화가 어려움
- 일부 항목은 시험성적서, 인증서, 시공 확인서 등 별도 문서가 필요함
- 최종 인증 판단은 전문가 검토가 필요함

추후 개선 방향은 다음과 같습니다.

- 표 구조 인식 고도화
- 문서별 전용 extractor 확장
- OCR 결과와 원문 위치를 연결한 evidence viewer 구현
- BIM/CAD 데이터 연동
- 인증 신청 문서 초안 자동 생성
- 사용자 피드백을 반영하는 검토 이력 관리 기능 추가

## 📝 프로젝트 성격

이 프로젝트는 완전 자율형 Agent라기보다, G-SEED 사전검토 업무를 단계별로 지원하는 문서 기반 AI Agent 프로토타입입니다.  
문서 분석, 기준 매핑, 점수 판정, 사람 검토, 개선안 추천, 질의응답을 하나의 흐름으로 연결하는 데 초점을 두었습니다.
